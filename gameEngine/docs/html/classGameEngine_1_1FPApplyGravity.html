GameEngine::FPApplyGravity::FPApplyGravity ( )
inline

Constructs the Death system and registers its component requirements.

During construction, the system declares that it requires the Health component. The ECS registry will only activate this system when at least one entity with a Health component exists.

Postcondition
System is configured to process entities with Health components.
The system is ready for scheduling within the Registry.

int GameEngine::FPApplyGravity::updateCount = 0

Counter tracking system execution calls.

Increments each time onUpdate() is called. Useful for:

  • Performance profiling
  • Unit testing (verifying the system runs)
  • Debugging system execution order
Remarks
Value persists throughout the system's lifetime. Manually reset if needed for test isolation.
