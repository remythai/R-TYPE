12/**
13 * @class Death
14 * @brief System that removes entities with zero health from the simulation.
15 *
16 * The Death system iterates through all entities that possess a Health
17 * component and automatically destroys any entity whose `currentHp` reaches
18 * zero. This ensures that dead entities are cleaned up from the ECS registry
19 * automatically each frame.
20 *
21 * @details
22 * This system is part of the damage and death handling pipeline:
23 * 1. Other systems reduce entity health (e.g., DamageSystem)
24 * 2. Death system checks for zero health each update
25 * 3. Dead entities are removed from the simulation
26 *
27 * This approach decouples the death logic from damage dealing, allowing
28 * flexible death handling without tight coupling.
29 *
30 * @note An entity is only destroyed if `currentHp == 0` (exact equality).
31 * This allows for potential "undead" mechanics where health might go
32 * negative.
33 *
34 * @performance
35 * - Time Complexity: O(n) where n is the number of entities with Health
36 * components
37 * - Space Complexity: O(1) - only iteration, no additional storage
38 *
39 * @see Health
40 * @see Registry::each
41 * @see Registry::destroy
42 */
43class FPApplyGravity : public System<FPApplyGravity>

47 * @brief Constructs the Death system and registers its component

64 /**
65 * @brief Processes all entities with Health components and destroys those
66 * with zero HP.
67 *
68 * Called every frame by the Registry. This method:
69 * 1. Increments the update counter for profiling/debugging
70 * 2. Iterates through all entities with a Health component
71 * 3. Checks each entity's `currentHp` value
72 * 4. Destroys any entity with exactly zero health
73 *
74 * @param registry Reference to the main ECS Registry for entity management.
75 * @param dt Delta time since last update in seconds. Unused but provided
76 * for consistency with the System interface.
77 *
78 * @attention Destroying entities during iteration is safe because the ECS
79 * system uses optimized pool iteration that doesn't invalidate cursors.
80 *
81 * @warning If an entity's health goes below zero (negative HP), it will NOT
82 * be destroyed by this system. Ensure damage systems clamp health to 0.
83 *
84 * @example
85 * ```cpp

88 *
89 * // If entity `e` has Health with currentHp = 0:

92 *
93 * @see Registry::each
94 * @see Registry::destroy
95 * @see Health::currentHp
96 */

107 }
108 });
109 }
