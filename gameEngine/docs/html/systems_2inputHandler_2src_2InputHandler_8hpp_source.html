15 /**

17 * @brief System responsible for translating player input into entity actions and motion.
18 *
19 * The `InputHandlerSystem` processes all entities that have:
20 * - `InputControlled` — tracks current input commands.
21 * - `Acceleration` — modified to apply directional movement.
22 *
23 * Depending on the inputs, the system:
24 * - Updates the entity’s acceleration to move it up, down, left, or right.
25 * - Handles shooting by spawning a projectile entity with the appropriate components
26 * (`Health`, `Damage`, `Velocity`, `Acceleration`, and `Position`).
27 *
28 * This system represents the bridge between player input and in-game motion/behavior.
29 */

39 InputHandlerSystem() {

43 /**
44 * @brief Processes input for all entities and updates their acceleration or triggers actions.
45 *
46 * For each `InputControlled` entity:
47 * - Resets acceleration to zero.
48 * - Applies directional acceleration based on pressed keys.
49 * - If the "shoot" input is detected, spawns a new projectile entity.

52 * - `Health(1, 1)`

71 GameEngine::Position playerPos{};
