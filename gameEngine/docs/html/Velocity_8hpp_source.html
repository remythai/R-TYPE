6/**
7 * @struct Velocity
8 * @brief Component representing an entity's movement speed with maximum
9 * constraints.
10 *
11 * Stores current velocity in pixels per frame along X and Y axes,
12 * plus a maximum speed limit to prevent infinite acceleration.
13 *
14 * @details
15 * **Velocity Mechanics:**
16 * - `x, y`: Current movement speed (pixels per frame)
17 * - `speedMax`: Upper limit for velocity magnitude
18 * - Modified by Acceleration component each frame
19 * - Used by Motion system to update Position
20 *
21 * **Physics Pipeline:**
22 * 1. Deceleration reduces velocity (friction)
23 * 2. Acceleration adds force to velocity
24 * 3. Velocity clamped to [-speedMax, +speedMax]
25 * 4. Position updated by velocity amount
26 *
27 * **Direction Convention:**
28 * - Positive X: Rightward movement
29 * - Negative X: Leftward movement
30 * - Positive Y: Downward movement (screen coords)
31 * - Negative Y: Upward movement
32 *
33 * **Speed Limiting:**
34 * ```cpp
35 * // Ensures velocity never exceeds bounds:
36 * vel.x = clamp(vel.x, -vel.speedMax, vel.speedMax);
37 * vel.y = clamp(vel.y, -vel.speedMax, vel.speedMax);
38 * ```
39 *
40 * **Typical Speed Values:**
41 * - Slow (tanks, heavy units): 1-3 pixels/frame
42 * - Normal (players, enemies): 5-10 pixels/frame
43 * - Fast (projectiles, dashing): 15-30 pixels/frame
44 * - Very fast (lasers, missiles): 50+ pixels/frame
45 *
46 * @inherits Component<Velocity> for ECS registration.
47 *
48 * @performance
49 * - Clamping occurs every frame in Motion system
50 * - Speed limit prevents physics instability
51 * - Deceleration creates smooth stopping behavior
52 *
53 * @example
54 * ```cpp
55 * // Player with moderate speed limit
56 * player.add<Velocity>(8.0f, 0.0f, 0.0f); // Max 8 px/frame, currently still
57 *
58 * // Fast-moving bullet
59 * bullet.add<Velocity>(20.0f, 15.0f, 0.0f); // Max 20 px/frame, moving right
60 *
61 * // Slow enemy patrol
62 * enemy.add<Velocity>(3.0f, -2.0f, 0.0f); // Max 3 px/frame, moving left
63 * ```
64 *
65 * @example
66 * ```cpp
67 * // Diagonal movement (combining X and Y):
68 * auto& vel = entity.get<Velocity>();
69 * vel.x = 5.0f; // Move right
70 * vel.y = -5.0f; // Move up
71 * // Resulting movement: diagonal up-right at ~7.07 px/frame
72 * ```
73 *
74 * @example
75 * ```cpp
76 * // Dynamic speed boost power-up:
77 * void applySpeedBoost(Entity entity, float multiplier) {
78 * auto& vel = entity.get<Velocity>();
79 * vel.speedMax *= multiplier; // Temporary speed increase
80 * // Remember to restore original speedMax after duration
81 * }
82 * ```
83 *
84 * @note
85 * - speedMax is per-axis limit, not total magnitude
86 * - Diagonal movement can exceed speedMax (√2 × speedMax)
87 * - Zero velocity with speedMax > 0 allows future acceleration
88 * - Constructor order: speedMax first, then current velocities
89 *
90 * @attention
91 * - speedMax = 0 prevents all movement
92 * - Very high speedMax may cause tunneling (passing through walls)
93 * - Negative speedMax creates undefined behavior
94 *
95 * @see Acceleration
96 * @see Position
97 * @see Motion
98 */
