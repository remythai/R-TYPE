11/**
12 * @class Animation
13 * @brief System that manages sprite animation by cycling through texture
14 * rectangles.
15 *
16 * The Animation system handles frame-based sprite animations by:
17 * 1. **Timing**: Tracks elapsed time since system initialization
18 * 2. **Frame calculation**: Determines current animation frame based on time
19 * 3. **Rect updates**: Updates the Renderable component's current texture
20 * rectangle
21 *
22 * This enables smooth sprite animations without per-entity timers.
23 *
24 * @details
25 * **Animation Pipeline (per frame):**
26 * 1. **Time Calculation**: Computes milliseconds since system start
27 * 2. **Frame Selection**: Uses modulo arithmetic to cycle through frames
28 * 3. **Rect Update**: Sets currentRectPos to the appropriate frame
29 *
30 * **Frame Selection Formula:**
31 * ```
32 * frameIndex = (elapsedMs / frameDuration) % totalFrames
33 * ```
34 * This creates a looping animation that repeats indefinitely.
35 *
36 * **Timing Behavior:**
37 * - All entities share the same time reference (startPoint)
38 * - Synchronizes animations across multiple entities
39 * - Frame duration determines animation speed (ms per frame)
40 *
41 * @requires
42 * - Renderable: Must contain non-empty rectPos vector for animation frames
43 * - frameDuration: Must be set to control animation speed
44 *
45 * @performance
46 * - Time Complexity: O(n) where n is number of animated entities
47 * - Space Complexity: O(1) per entity
48 * - Single time query per frame for all entities
49 *
50 * @note
51 * - Entities with empty rectPos vectors are skipped
52 * - All animations start synchronized at system creation
53 * - Frame duration is in milliseconds
54 *
55 * @example
56 * ```cpp
57 * // Entity with 4 animation frames, 100ms per frame
58 * // rectPos = [rect0, rect1, rect2, rect3]
59 * // frameDuration = 100
60 * // At 250ms: frame = (250 / 100) % 4 = 2 â†’ displays rect2
61 * ```
62 *

64 * @see Registry
65 */
