11/**
12 * @struct Renderable
13 * @brief Component containing all data necessary for sprite rendering and
14 * animation.
15 *
16 * Stores sprite sheet information, texture coordinates, animation frames,
17 * and screen dimensions. Central component for the rendering system.
18 *
19 * @details
20 * **Rendering Pipeline:**
21 * 1. **Sprite Sheet**: Loads texture from spriteSheetPath
22 * 2. **Frame Selection**: Uses currentRectPos to select texture region
23 * 3. **Size Definition**: rectSize defines the sprite dimensions
24 * 4. **Animation**: Animation system cycles through rectPos frames
25 *
26 * **Texture Coordinates:**
27 * - `currentRectPos`: Top-left corner of current frame in sprite sheet
28 * - `rectPos`: Vector of all animation frame positions
29 * - `rectSize`: Width and height of each frame
30 * - Coordinates in pixels within the sprite sheet texture
31 *
32 * **Animation Control:**
33 * - `autoAnimate`: If true, Animation system processes this entity
34 * - `frameDuration`: Milliseconds per animation frame
35 * - `rectPos.size()`: Number of frames in animation loop
36 *
37 * **Screen Boundaries:**
38 * - `screenSizeX/Y`: Used by Motion system for position clamping
39 * - Prevents entities from moving off-screen
40 * - Should match actual window/viewport dimensions
41 *
42 * **Sprite Sheet Format:**
43 * ```
44 * [Frame0][Frame1][Frame2]
45 * [Frame3][Frame4][Frame5]
46 * ```
47 * Each frame position specified as vec2(x, y) in rectPos.
48 *
49 * @inherits Component<Renderable> for ECS registration.
50 *
51 * @performance
52 * - Move semantics used for spriteSheetPath and rectPos
53 * - No dynamic allocations during rendering
54 * - Texture loaded once and cached by rendering system
55 *
56 * @example
57 * ```cpp
58 * // Static sprite (no animation)
59 * entity.add<Renderable>(
60 * 1920, 1080, // Screen size
61 * "assets/player.png", // Sprite sheet path
62 * {{0, 0}}, // Single frame at origin
63 * vec2(32, 32), // 32x32 pixel sprite
64 * 0, // No frame duration needed
65 * false // No auto-animation
66 * );
67 * ```
68 *
69 * @example
70 * ```cpp
71 * // Animated walking cycle (4 frames, 100ms each)
72 * entity.add<Renderable>(
73 * 1920, 1080, // Screen size
74 * "assets/walk_cycle.png", // Sprite sheet
75 * { // 4 animation frames
76 * vec2(0, 0), vec2(32, 0),
77 * vec2(64, 0), vec2(96, 0)
78 * },
79 * vec2(32, 48), // 32x48 sprite size
80 * 100, // 100ms per frame = 10 FPS
81 * true // Enable auto-animation
82 * );
83 * ```
84 *
85 * @example
86 * ```cpp
87 * // Multi-row sprite sheet
88 * std::vector<vec2> frames;
89 * for (int row = 0; row < 2; ++row) {
90 * for (int col = 0; col < 4; ++col) {
91 * frames.push_back(vec2(col * 64, row * 64));
92 * }
93 * }
94 * entity.add<Renderable>(
95 * 1920, 1080,
96 * "assets/character.png",
97 * frames, // 8 frames total
98 * vec2(64, 64),
99 * 150,
100 * true
101 * );
102 * ```
103 *
104 * @note
105 * - Empty rectPos creates non-renderable entity
106 * - currentRectPos auto-set by Animation system if autoAnimate = true
107 * - Manually set currentRectPos for state-based animations (idle/walk/jump)
108 * - screenSizeX/Y should be consistent across all Renderable components
109 *
110 * @attention
111 * - spriteSheetPath must be valid; missing files cause rendering errors
112 * - All frames in rectPos should fit within sprite sheet bounds
113 * - frameDuration = 0 with autoAnimate = true causes division by zero
114 *
115 * @see Animation
116 * @see Position
117 * @see Motion
118 * @see vec2
119 */
