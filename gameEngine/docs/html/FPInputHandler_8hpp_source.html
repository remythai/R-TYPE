19/**
20 * @class FPInputHandler
21 * @brief System that processes player input and manages input-controlled
22 * entities.
23 *
24 * The InputHandler system translates input commands into entity behavior by:
25 * 1. Reading input states from InputControlled components
26 * 2. Updating entity velocity based on directional inputs
27 * 3. Creating projectiles when shoot input is detected
28 *
29 * This system is responsible for converting raw input into game mechanics,
30 * serving as the bridge between player input and entity physics/behavior.
31 *
32 * @details
33 * **Input Mapping:**
34 * - Input 0: Move down (positive Y velocity)
35 * - Input 1: Move up (negative Y velocity)
36 * - Input 2: Move left (negative X velocity)
37 * - Input 3: Move right (positive X velocity)
38 * - Input 4: Shoot (creates projectile entity)
39 *
40 * **Projectile Creation:**
41 * When input 4 (shoot) is detected, the system:
42 * 1. Creates a new entity
43 * 2. Attaches required components (Renderable, Health, Damage, etc.)
44 * 3. Positions it at the player's current position
45 * 4. Sets up physics (Velocity, Velocity)
46 * 5. Configures collision (Collider with bitset)
47 * 6. Restricts movement within domain boundaries
48 *
49 * @note Multiple inputs can be processed in a single frame, allowing
50 * simultaneous movement and shooting.
51 *
52 * @requires
53 * - InputControlled: Provides input commands
54 * - Velocity: Modified by input
55 * - Renderable: Required for entity visibility (used for requirement checking)
56 * - Position: Used when creating projectiles
57 *
58 * @see InputControlled
59 * @see Velocity
60 * @see Position
61 * @see Renderable
62 * @see Collider
63 * @see Domain
64 */
