6/**
7 * @struct OnPickup
8 * @brief Component defining power-up effects applied when entity is collected.
9 *
10 * Stores stat modifications and buffs that are transferred to the collecting
11 * entity. Used for pickups, collectibles, and temporary power-ups in games.
12 *
13 * @details
14 * **Pickup Effects:**
15 * - **Instant bonuses**: Applied immediately on pickup
16 * - `hpBonus`: Heal amount (can be negative for damage pickups)
17 * - `hpMaxBonus`: Permanent max health increase
18 * - **Temporary buffs**: Applied for limited duration
19 * - `dmgBonus`: Increase attack damage
20 * - `cooldownBonus`: Reduce ability cooldowns (negative = faster)
21 * - `scoreMultiplierBonus`: Increase points earned
22 * - `duration`: How long temporary buffs last (seconds)
23 *
24 * **Processing Workflow:**
25 * 1. Collision detected between player and pickup entity
26 * 2. OnPickup effects extracted from pickup entity
27 * 3. Instant bonuses applied immediately to player stats
28 * 4. Temporary buffs tracked with expiration timers
29 * 5. Pickup entity destroyed after collection
30 *
31 * **Bonus Stacking:**
32 * - Instant bonuses: Add/subtract from current values
33 * - Temporary buffs: Multiple pickups may stack or override
34 * - Duration: Each pickup may have independent timer
35 *
36 * @inherits Component<OnPickup> for ECS registration.
37 *
38 * @example
39 * ```cpp
40 * // Health pack: Restore 50 HP instantly
41 * healthPack.add<OnPickup>(50, 0, 0, 0.0f, 0.0f, 0.0f);
42 *
43 * // Max health upgrade: Permanently increase max HP by 25
44 * upgrade.add<OnPickup>(0, 25, 0, 0.0f, 0.0f, 0.0f);
45 *
46 * // Damage boost: +10 damage for 15 seconds
47 * damagePickup.add<OnPickup>(0, 0, 10, 0.0f, 0.0f, 15.0f);
48 *
49 * // Score multiplier: 2x points for 20 seconds
50 * multiplier.add<OnPickup>(0, 0, 0, 0.0f, 1.0f, 20.0f);
51 *
52 * // Rapid fire: -0.5s cooldown for 10 seconds
53 * rapidFire.add<OnPickup>(0, 0, 0, -0.5f, 0.0f, 10.0f);
54 *
55 * // Super combo: Multiple effects for 30 seconds
56 * superPickup.add<OnPickup>(100, 0, 20, -0.3f, 0.5f, 30.0f);
57 * ```
58 *
59 * @example
60 * ```cpp
61 * // Typical pickup processing system:
62 * void onCollision(Entity player, Entity pickup) {
63 * auto& onPickup = pickup.get<OnPickup>();
64 * auto& health = player.get<Health>();
65 *
66 * // Apply instant bonuses
67 * health.currentHp += onPickup.hpBonus;
68 * health.maxHp += onPickup.hpMaxBonus;
69 *
70 * // Register temporary buffs
71 * if (onPickup.duration > 0) {
72 * registerBuff(player, onPickup);
73 * }
74 *
75 * // Remove pickup entity
76 * registry.destroy(pickup);
77 * }
78 * ```
79 *
80 * @note
81 * - Duration of 0 means instant-only effects (no temporary buffs)
82 * - Negative bonuses can create hazardous pickups (curses)
83 * - Consider clamping health bonuses to prevent overflow
84 * - scoreMultiplierBonus is additive (1.0 = +100% = 2x multiplier)
85 *
86 * @see Health
87 * @see Damage
88 * @see Collider
89 */
