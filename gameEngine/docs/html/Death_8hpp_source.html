10/**
11 * @class Death
12 * @brief System that removes entities with zero health from the simulation.
13 *
14 * The Death system iterates through all entities that possess a Health
15 * component and automatically destroys any entity whose `currentHp` reaches
16 * zero. This ensures that dead entities are cleaned up from the ECS registry
17 * automatically each frame.
18 *
19 * @details
20 * This system is part of the damage and death handling pipeline:
21 * 1. Other systems reduce entity health (e.g., DamageSystem)
22 * 2. Death system checks for zero health each update
23 * 3. Dead entities are removed from the simulation
24 *
25 * This approach decouples the death logic from damage dealing, allowing
26 * flexible death handling without tight coupling.
27 *
28 * @note An entity is only destroyed if `currentHp == 0` (exact equality).
29 * This allows for potential "undead" mechanics where health might go
30 * negative.
31 *
32 * @performance
33 * - Time Complexity: O(n) where n is the number of entities with Health
34 * components
35 * - Space Complexity: O(1) - only iteration, no additional storage
36 *
37 * @see Health
38 * @see Registry::each
39 * @see Registry::destroy
40 */

67 * 3. Checks each entity's `currentHp` value
68 * 4. Destroys any entity with exactly zero health
